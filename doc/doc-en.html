Documentation for the auto-documenting system Documentor

Using programm

Using in command line

Type at the command line: documentor ProjectName ListFiles
As a result, you will get 2 generated HTML and CSS documentation files.
Make sure your PATH variable is set

Getting help

For getting help enter: documentor /?

Set config file

Set language of documenting

For set the language the LANGUAGE parameter must be assigned one of two values: RUS or ENG.(Default - RUS)

Set the color-scheme of the document

For set the color-scheme of the document use 9 parameters:

WARNING
Warning color
BACKGROUND
Background color
SEPARATOR
Separator color
MENUBORDER
Border of menu color
MENUSELECT
Color of selected element
MENU
Menu color
TABLE1
Color of odd table rows
TABLE2
Color of even table rows
TABLEBORDER
Table border color

The color value can be specified in any form allowed by the HTML standard.

Setting the size of buffers storing information about the HTML representation of the documented object

You can adjust the size for all 9 types of documentable objects:

  1. SIZEMODULEHTML
  2. SIZEFUNCHTML
  3. SIZEGROUPFUNCHTML
  4. SIZEVARHTML
  5. SIZESTRUCTHTML
  6. SIZEENUMHTML
  7. SIZEUNIONHTML
  8. SIZEMACROHTML
  9. SIZEGROUPMACROHTML

Documenting a code

Documenting a module

You can document the module by adding a comment in the file that looks like this with the file keyword:
/*\file
text
*/
 The keyword must be on the same line as the beginning of the comment

Documenting a function

You can document the module by adding a comment in the file that looks like this with the function keyword:
/*\function funcname
text
\param param1 description
\param param2 description
\result description
*/
 Keywords must go exactly in this order
Descriptions other than the main one must be one-line

Documenting a group of function

You can document a group of functions by adding a comment in the file that looks like this with the funcgroup keyword:
/*\funcgroup funcgroupname
text
\param param1 description
\param param2 description
\result description
*/
/*begingroup*/
list of function's definitions
/*endgroup*/
 The comment / * begingroup * / must be below the main description

Documenting a variable

You can document the variable by adding the following comment in the file with the var keyword:
/*\var varname
text
*/

Documenting a struct, enum or union

This can be documented by adding comments of the following type in the file above the structure with the keyword struct, enum or union:
/*\struct structname
text
*/
struct anystruct{
int member1; /*Description*/
char member2; /*Description*/
}

Documenting a macro

You can document the macro by adding comments of the following type with the macro keyword in the file above the macro:
/*\macro macroname
text
*/

Documenting a group of macro

You can document a group of macros by adding comments of the following type in the file with the macrogroup keyword:
/*\macrogroup groupname
text
*/
/*begingroup*/
list of macro's definitions
/*endgroup*/

Adding warnings to the documentation

You can add a warning by writing text and wrapping it in span tags.
