Документация к системе автодокументации кода Documentor

Настройка файла конфигурации

Настройка языка документирования

Для настройки языка параметру LANGUAGE нужно присвоить одно из двух значений: RUS или ENG.(По умолчанию - RUS)

Настройка цветовой схемы документа

Для настройки цветовой схемы используеться 9 параметров:

WARNING
Цвет текста предупреждения
BACKGROUND
Цвет фона
SEPARATOR
Цвет разделителя
MENUBORDER
Цвет границы элемента меню
MENUSELECT
Цвет выбранного элемента выпадающего меню
MENU
Общий цвет меню
TABLE1
Цвет нечётных строк таблицы
TABLE2
Цвет чётных строк таблицы
TABLEBORDER
Цвет границы таблицы

Значение цвета можно указывать в любом допустимом стандартом HTML виде.

Настройка размера буферов хранящих информацию об HTML представлении документируемого объекта

Настраивать размер можно для всех 9 типов документируемых объектов:

  1. SIZEMODULEHTML
  2. SIZEFUNCHTML
  3. SIZEGROUPFUNCHTML
  4. SIZEVARHTML
  5. SIZESTRUCTHTML
  6. SIZEENUMHTML
  7. SIZEUNIONHTML
  8. SIZEMACROHTML
  9. SIZEGROUPMACROHTML

Документирование кода

Документирование модуля

Задокументировать модуль можно добавив в файле комментарий следующего вида с ключевым словом file:
/*\file
text
*/
 Ключевое слово обязательно должно быть на той же строке, что и начала комментария

Документирование функции

Задокументировать функцию можно добавив в файле комментарий следующего вида с ключевым словом function:
/*\function funcname
text
\param param1 description
\param param2 description
\result description
*/
 Ключевые слова должны идти именно в такой последовательности
Описания кроме основного должны быть однострочными

Документирование группы функций

Задокументировать группу функций можно добавив в файле комментарий следующего вида с ключевым словом funcgroup:
/*\funcgroup funcgroupname
text
\param param1 description
\param param2 description
\result description
*/
/*begingroup*/
list of function's definitions
/*endgroup*/
 Комментарий /*begingroup*/ должен быть нижеосновного описания

Документирование переменной

Задокументировать переменную можно добавив в файле комментарий следующего вида с ключевым словом var:
/*\var varname
text
*/

Документирование структуры, перечисления или объединения

Задокументировать это можно добавив в файле над структурой комментарии следующего вида с ключевым словом struct, enum или union:
/*\struct structname
text
*/
struct anystruct{
int member1; /*Description*/
char member2; /*Description*/
}

Документирование макроса

Задокументировать макрос можно добавив в файле над макросом комментарии следующего вида с ключевым словом macro:
/*\macro macroname
text
*/

Документирование группы макросов

Задокументировать группу макросов можно добавив в файле комментарии следующего вида с ключевым словом macrogroup:
/*\macrogroup groupname
text
*/
/*begingroup*/
list of macro's definitions
/*endgroup*/

Добавление предупреждений в документацию

Добавить предупреждение можно написав текст и обернув его в теги span.
